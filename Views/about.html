Node modules we have used for this project:

HTTP The http module in Node.js enables the creation of web servers and handling HTTP requests and responses. It is essential for building web applications and APIs.
With this module, you can create servers that listen on specific ports, handle incoming requests, and send responses back to clients. Additionally, it allows making HTTP requests to other servers.


File System (fs) The fs module provides an API for interacting with the file system, allowing you to read from, write to, and manipulate files and directories.
It supports both synchronous and asynchronous operations for reading and writing files. The module also enables manipulation of file paths and directories and can monitor changes in the file system.


Path The path module in Node.js offers utilities for working with file and directory paths in a platform-independent manner. This is particularly useful when dealing with file systems across different operating systems.
The module allows you to join different segments of a path to create a valid path string, resolve absolute paths, extract parts of a path such as directory names, file names, and extensions, and normalize paths to remove redundant segments.


Events The events module in Node.js, centered around the EventEmitter class, enables an event-driven architecture. It allows developers to create, emit, and handle custom events using methods like emit and on.
This is essential for building scalable, asynchronous applications where components react to events efficiently.
